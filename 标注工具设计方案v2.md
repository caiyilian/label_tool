# 坐姿标注工具设计方案 v2

## 1. 目标

开发一个 Web 端标注工具，对已抽好的帧图片进行逐帧标注。每个学生在每帧中需要标注的最终字段如下：

| 字段 | 类型 | 取值 |
|------|------|------|
| `student_id` | 文本 | 匿名编号，如 `S01`、`S02`，同一学生在整段视频中保持一致 |
| `bbox` | 坐标 | `[x1, y1, x2, y2]`，该学生在画面中的人体检测框 |
| `status` | 单选 | `sitting_listening` / `sitting_reading` / `standing` / `left_seat` |
| `posture` | 单选 | `normal` / `bad` |
| `bad_reason` | 多选 | `head_tilt` / `head_down` / `lean_forward` / `side_lean`（仅当 `posture=bad` 时填写） |
| `pose_keypoints` | 点位数组 | 8 个上半身关键点，用于辅助姿态判断（详见 §13） |

---

## 2. 核心问题：如何选中一个学生？

### 2.1 结论：用框（BBox）

经过对比三种方案，**采用画框（Bounding Box）作为选中学生的方式**。

| 方案 | 优点 | 缺点 | 结论 |
|------|------|------|------|
| **画框 (BBox)** | 直观、可同时作为检测标签、划定区域后标签面板自动弹出 | 每帧每个学生都要画，工作量中等 | **采用** |
| 点击标记 (Point) | 速度快 | 歧义大（密集座位点不准）、无法提供检测框给模型 | 不采用 |
| 先做姿态估计再分类 | 信息最丰富 | 技术门槛高、标注速度极慢、当前阶段不需要 | 不采用 |

### 2.2 理由

1. **框 = 选中 + 标签一体**：画完框后该学生即被"选中"，自动弹出标签面板，不需要额外的选中操作。
2. **框是模型训练的必要输入**：目标检测模型（如 YOLO）需要 bbox 作为监督标签，画框一举两得。
3. **框内区域可裁剪**：后续做姿态估计或精细分类时，可直接裁剪框内区域作为输入。
4. **框可辅助遮挡判断**：标注员看到框与框重叠时，可直接判断遮挡程度。

## 3. 标注工作流

### 3.1 整体流程

```
加载一帧图片
    ↓
标注员对该帧中每个可见学生依次执行：
    ① 画框 → ② 自动分配 student_id → ③ 标注姿态点位（可选）
    → ④ 填写分类标签 → ⑤ 确认
    ↓
当前帧所有学生标注完毕 → 点击"下一帧" → 重复
```

### 3.2 单个学生的标注步骤（详细）

```
步骤 1：画框
    鼠标在画面上拖拽，画出一个矩形框，框住该学生的身体区域。
    框的范围：从头顶到臀部下方，尽量贴合身体轮廓。

步骤 2：分配 student_id
    系统自动分配新编号（S01, S02, ...），或从下拉列表选择已有编号。
    同一编号在整段视频中指向同一个学生。

步骤 3：标注姿态点位（可选）
    画框后，可选进入点位标注模式，依次点击放置 8 个关键点：
    头 → 脖子 → 右肩 → 右肘 → 右手 → 左肩 → 左肘 → 左手
    点位可拖拽调整，被遮挡的点可标记为"不可见"。
    如不标注点位，直接跳到步骤 4。

步骤 4：填写标签面板
    ┌─────────────────────────────────┐
    │  学生: [S01 ▾]                  │
    │                                 │
    │  状态:                          │
    │  ○ 坐-听说  ○ 坐-读写           │
    │  ○ 站立     ○ 离席              │
    │                                 │
    │  坐姿:                          │
    │  ○ 正常     ○ 不良              │
    │                                 │
    │  不良原因（多选，仅坐姿=不良时）:  │
    │  □ 歪头  □ 低头  □ 俯身  □ 侧身 │
    │                                 │
    │  姿态点位: 已标 6/8  [清除] [重标]│
    │                                 │
    │  备注: [________]               │
    │                                 │
    │  [确认]  [删除此框]              │
    └─────────────────────────────────┘

步骤 5：确认
    点击"确认"后，该学生的框和骨架在画面上显示。
    点击已有标注框可重新编辑。
```

### 3.3 前后视角的处理方式

标注以**前视角（front）为主视角**，后视角（back）仅作为参考，不做标注。

```
┌──────────────────────┬──────────────────────┐
│                      │                      │
│    前视角 (front)     │    后视角 (back)      │
│    【主标注区域】      │    【参考区域】        │
│                      │                      │
│    画框 + 点位 + 标签  │    仅查看，辅助判断    │
│                      │    不做任何标注        │
│                      │                      │
└──────────────────────┴──────────────────────┘
```

训练时的输入：前视角视频 + 后视角视频 + 标注（仅前视角产生）。
后视角为模型提供多角度信息，但不需要人工标注后视角的 bbox 或标签。

## 4. 快捷操作设计

为提升标注效率，设计以下快捷键和交互：

| 操作 | 方式 | 说明 |
|------|------|------|
| 画框 | 鼠标拖拽 | 在前视角画面上按住左键拖拽画矩形 |
| 编辑标签 | 双击已有框 | 双击已标注的框，弹出标签面板供修改 |
| 标注点位 | 选中框后按 `K` | 进入点位标注模式，依次点击放置 8 个关键点 |
| 删除标注 | 右键菜单 或 面板按钮 | 右键点击框 → 删除，或在面板中点击"删除此框" |
| 复制上一帧 | 快捷键 `C` | 将上一帧的所有标注复制到当前帧，然后微调 |
| 确认并下一个 | `Enter` | 确认当前标签面板，关闭面板 |
| 撤销 | `Ctrl+Z` | 撤销上一步操作 |
| 重做 | `Ctrl+Y` | 重做上一步撤销的操作 |
| 上一帧 / 下一帧 | `←` / `→` | 在帧之间导航 |
| 重置视图 | `R` | 恢复图片初始位置和大小 |
| 缩放平移 | 滚轮缩放 + 中键拖拽 | 与现有抽帧工具的预览交互一致 |

## 5. 标签面板的智能行为

### 5.1 状态联动

```
当 status = "left_seat" 时：
    → posture 自动置灰，强制为 "normal"（离席不需要判断坐姿）
    → bad_reason 全部清空并置灰

当 status = "standing" 时：
    → posture 可选（站姿也可判断是否不良，如驼背）
    → bad_reason 仍然可用

当 status = "sitting_listening" 或 "sitting_reading" 时：
    → posture 正常可选
    → 当 posture = "normal" 时，bad_reason 自动清空并置灰
    → 当 posture = "bad" 时，bad_reason 解锁，至少选一项
```

### 5.2 默认值

| 字段 | 默认值 |
|------|--------|
| status | `sitting_listening` |
| posture | `normal` |
| bad_reason | 空（无） |
| pose_keypoints | 空（未标注） |

## 6. 帧导航与批量标注策略

### 6.1 帧导航

- 抽帧工具输出的图片按 `front_000001.jpg`, `front_000002.jpg`, ... 命名。
- 标注工具按文件名排序加载帧序列。
- 提供帧列表侧边栏，可快速跳转到任意帧。
- 已完成标注的帧在列表中打勾标记。

### 6.2 批量复制

- **复制上一帧（核心功能）**：课堂场景中学生位置和姿态变化缓慢，大部分帧与上一帧相同。标注员只需复制上一帧，然后微调有变化的学生。
- 复制时，框、标签、点位全部复制。
- 这是提升效率的最关键功能。

## 7. 标注数据存储格式

### 7.1 每帧存储为一个 JSON 文件

文件命名：`front_000001.json`（与对应图片同名，后缀改为 `.json`）

```json
{
  "frame_id": "front_000001",
  "image_file": "front_000001.jpg",
  "annotations": [
    {
      "student_id": "S01",
      "bbox": [480, 220, 535, 390],
      "status": "sitting_reading",
      "posture": "bad",
      "bad_reason": ["head_down", "lean_forward"],
      "pose_keypoints": [
        { "name": "head",          "x": 505, "y": 225, "visible": true },
        { "name": "neck",          "x": 508, "y": 248, "visible": true },
        { "name": "right_shoulder","x": 492, "y": 260, "visible": true },
        { "name": "right_elbow",   "x": 478, "y": 295, "visible": true },
        { "name": "right_hand",    "x": 470, "y": 330, "visible": false },
        { "name": "left_shoulder", "x": 522, "y": 258, "visible": true },
        { "name": "left_elbow",    "x": 535, "y": 290, "visible": true },
        { "name": "left_hand",     "x": 540, "y": 325, "visible": true }
      ],
      "note": ""
    },
    {
      "student_id": "S02",
      "bbox": [620, 240, 680, 410],
      "status": "sitting_listening",
      "posture": "normal",
      "bad_reason": [],
      "pose_keypoints": [],
      "note": ""
    }
  ]
}
```

### 7.2 关键点定义

| 序号 | name | 中文 | 说明 |
|------|------|------|------|
| 0 | `head` | 头 | 头顶正中（发际线附近） |
| 1 | `neck` | 脖子 | 颈部与躯干连接处 |
| 2 | `right_shoulder` | 右肩 | 右侧肩关节 |
| 3 | `right_elbow` | 右肘 | 右侧肘关节 |
| 4 | `right_hand` | 右手 | 右侧手腕或手掌 |
| 5 | `left_shoulder` | 左肩 | 左侧肩关节 |
| 6 | `left_elbow` | 左肘 | 左侧肘关节 |
| 7 | `left_hand` | 左手 | 左侧手腕或手掌 |

骨架连接关系（从脖子出发的树形结构）：

```
        head
         |
        neck
       /    \
  R.shoulder  L.shoulder
      |          |
  R.elbow     L.elbow
      |          |
  R.hand      L.hand
```

### 7.3 可见性说明

每个关键点的 `visible` 字段：
- `true`：可见，坐标有效
- `false`：被遮挡或不在画面中，坐标为最后估计位置（或 0,0），训练时忽略

### 7.4 学生注册表（全局）

文件命名：`students.json`，存储在输出目录根下。

```json
{
  "class_id": "Grade7_Class02",
  "students": [
    { "student_id": "S01", "seat_row": 3, "seat_col": 2 },
    { "student_id": "S02", "seat_row": 3, "seat_col": 3 }
  ]
}
```

### 8.2 点位标注模式

按 `K` 或点击"进入点位标注"后，进入点位标注模式：

```
┌─────────────────────────────────────────────────────────────────┐
│  点位标注模式 - S01                    [退出点位标注 (Esc)]       │
├─────────────────────────────────────────────────────────────────┤
│                                                                 │
│    当前需要标注: 脖子 (2/8)                                      │
│                                                                 │
│    点击画面上的 脖子 位置                                         │
│    右键 = 跳过此点（标记为不可见）                                 │
│    Esc = 退出点位标注模式                                         │
│                                                                 │
│    进度: ●●○○○○○○                                               │
│    head ✓ → neck ← → R.shoulder → R.elbow → R.hand              │
│              → L.shoulder → L.elbow → L.hand                     │
│                                                                 │
└─────────────────────────────────────────────────────────────────┘
```

点位标注规则：
- 进入模式后，按 head → neck → R.shoulder → R.elbow → R.hand → L.shoulder → L.elbow → L.hand 的顺序依次标注
- 左键点击 = 放置当前点，自动跳到下一个点
- 右键点击 = 跳过当前点（标记为 visible=false），自动跳到下一个点
- 全部标完后自动退出点位标注模式
- 已标注的点显示为绿色圆点，当前待标点显示为红色圆点
- 骨架连线实时预览

### 8.3 画面上的骨架显示

选中某个学生时，其框内会显示已标注的骨架：

```
        ○ head
         │
        ○ neck
       ╱    ╲
  ○ R.shoulder  ○ L.shoulder
      │          │
  ○ R.elbow     ○ L.elbow
      │          │
  ○ R.hand      ○ L.hand
```

- 点位为小圆点（半径 3~4px）
- 连线为细线（1~2px）
- 可见点为绿色，不可见点为灰色（不显示连线）
- 未标注点位的骨架不显示

## 9. 技术方案

| 层面 | 选型 | 理由 |
|------|------|------|
| 后端 | Flask (与现有工具复用) | 项目已有 Flask 基础，无需引入新框架 |
| 前端 | 原生 HTML/JS + Tailwind CSS | 与现有 Web 版风格一致，轻量无依赖 |
| 画框交互 | Canvas API | 原生 Canvas 绑定鼠标事件实现拖拽画框，性能好 |
| 点位标注 | Canvas API | 与画框共用同一 Canvas，点位作为特殊图形叠加渲染 |
| 图片加载 | 直接 `<img>` 加本地文件 | 标注工具在本地运行，图片路径直接访问 |
| 数据存储 | JSON 文件 | 简单直接，无需数据库，方便版本控制和后续处理 |

- 标注数据以 JSON 格式存储在 `annotations/` 子目录
- Toast 提醒：保存成功/失败以顶部浮动通知替代弹窗
- 空标注保护：无学生时不保存 JSON 文件，帧列表不标绿
- 文件夹自动重建：annotations 目录被删除后自动重建

*姿态点位标注（P3）*
- 8 个上半身关键点：头、脖子、右肩、右肘、右手、左肩、左肘、左手
- 连续标注模式（K 键）：按顺序依次标注，左键放置、右键跳过、全部完成自动退出
- 逐个编辑/删除：标签面板中每个点可单独"改"（进入该点标注模式）或"✕"（删除该点）
- 骨架绘制：画面上显示绿色圆点 + 连线，仅可见点和连线
- 点位数据与框、标签一起存储、加载、复制
