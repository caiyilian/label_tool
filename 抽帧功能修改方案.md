# 视频抽帧功能修改方案 — 可配置采样策略

## 1. 背景

### 1.1 当前实现

当前抽帧工具采用固定策略：**每秒抽取 1 帧**。

实现方式：使用 ffmpeg 的 `select` 滤镜，表达式为：
```
select='gte(n,{skip_frames})*not(mod(n-{skip_frames},{fps_round}))'
```

其中 `fps_round` 是视频帧率（通常 25fps），即每隔 25 帧取 1 帧，实现每秒 1 帧。

### 1.2 数据量估算

根据学校提供的信息：
- A 学校：25 个工作日 × 42 个教室
- B 学校：31 个工作日 × 24 个教室
- 每天有效时间段：8:00-12:00、14:00-17:00（共 7 小时）
- 每小时约 1 段视频

如果按每秒 1 帧：
- 总帧数 = (25×42 + 31×24) × 7 × 3600 ≈ **4500 万帧**
- 数据量过大，标注不现实

### 1.3 采样策略

采用间歇采样策略，参数可配置：
- **t1**：采样周期（分钟），每隔 t1 分钟启动一次采样窗口
- **t2**：采样窗口时长（分钟），每次采样持续 t2 分钟
- **t3**：窗口内采集间隔（秒），在采样窗口内每隔 t3 秒采集 1 帧

**约束条件**：
- t2 ≤ t1（窗口时长不能超过周期）
- t3 < t2（采集间隔必须小于窗口时长，不能等于）

### 1.4 数据量估算（新策略）

以当前默认参数 t1=5, t2=1, t3=10 为例：
- 每小时采样窗口数 = 60 / 5 = 12 次
- 每次窗口采集帧数 = 60 / 10 = 6 帧
- 每小时总帧数 = 12 × 6 = 72 帧
- 每天 7 小时 = 72 × 7 = 504 帧
- 总帧数 = (25×42 + 31×24) × 504 ≈ **90 万帧**

相比每秒 1 帧（4500 万帧），减少约 98% 的数据量。

### 2.2 后端修改（main.py）

#### 2.2.1 接收采样参数

在 `process_videos_thread` 函数中接收新参数：

```python
def process_videos_thread(data):
    v1 = data['v1']
    v2 = data['v2']
    out = data['out']
    t1_sec = data['t1_sec']
    t2_sec = data['t2_sec']
    res_limit = data.get('res_limit')
    # 新增采样参数
    sample_t1 = data.get('sample_t1', 5)  # 分钟
    sample_t2 = data.get('sample_t2', 1)  # 分钟
    sample_t3 = data.get('sample_t3', 10) # 秒
```

#### 2.2.2 修改 ffmpeg select 滤镜表达式

将原来的每秒 1 帧表达式：

```python
# 旧：每秒1帧
filter_str = f"select='gte(n,{skip_frames})*not(mod(n-{skip_frames},{fps_round}))'"
```

改为基于采样参数的表达式：

```python
# 新：可配置采样策略
window_period_frames = int(sample_t1 * 60 * fps_round)   # 采样周期（帧数）
window_duration_frames = int(sample_t2 * 60 * fps_round)  # 窗口时长（帧数）
frame_interval = int(sample_t3 * fps_round)               # 采集间隔（帧数）

filter_str = (
    f"select='"
    f"gte(n,{skip_frames})*"
    f"lt(mod((n-{skip_frames}),{window_period_frames}),{window_duration_frames})*"
    f"not(mod((n-{skip_frames}),{frame_interval}))"
    f"'"
)
```

**滤镜逻辑解释**：

对于跳过 `skip_frames` 帧后的每帧（相对帧号 = n - skip_frames）：

1. `gte(n, {skip_frames})`：确保跳过时间偏移的帧
2. `lt(mod((n-{skip_frames}), {window_period_frames}), {window_duration_frames})`：
   - 用取模运算判断当前帧是否在某个采样窗口内
   - 相对帧号对周期取模，如果小于窗口时长，则在窗口内
3. `not(mod((n-{skip_frames}), {frame_interval}))`：
   - 在窗口内，每隔 frame_interval 帧取 1 帧

#### 2.2.3 修改预期帧数计算

```python
# 旧：每秒1帧
expected_total_images = int((total_frames_in_video - skip_frames) / fps_round)

# 新：按采样策略计算
total_duration_sec = (total_frames_in_video - skip_frames) / fps_round
num_windows = int(total_duration_sec / (sample_t1 * 60))
frames_per_window = int(sample_t2 * 60 / sample_t3)
expected_total_images = num_windows * frames_per_window
if expected_total_images <= 0: expected_total_images = 1
```

#### 2.2.4 修改日志输出

```python
extract_log(f"采样策略: 每 {sample_t1} 分钟采样 {sample_t2} 分钟，每 {sample_t3} 秒取 1 帧")
extract_log(f"预期提取: {expected_total_images} 帧")
```

### 2.3 输出文件命名

输出文件名保持不变：`front_000001.jpg`, `front_000002.jpg`, ...

帧编号按实际抽取顺序递增，不体现采样时间。标注工具通过文件名中的编号进行前后视角配对。

## 6. 注意事项

### 6.1 ffmpeg 滤镜兼容性

ffmpeg 的 `select` 滤镜支持基本的数学运算（加减乘除、取模、比较），但不支持变量赋值。表达式需要直接计算所有值。

### 6.2 帧率处理

- 视频帧率可能不是整数（如 25.0、29.97、30.0 等）
- 需要将帧率四舍五入为整数用于计算
- 采样间隔的帧数 = t3 × round(fps)

### 6.3 时间偏移

- 两个视频的时间偏移处理保持不变
- 采样策略应用于偏移后的视频
- 跳过的帧数 = 时间偏移秒数 × 帧率

### 6.4 进度显示

- 进度条基于预期帧数计算
- ETA 基于已提取帧数和实际耗时估算
- 日志显示采样策略参数和预期帧数
