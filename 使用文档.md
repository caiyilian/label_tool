# 课堂行为分析工具使用文档

## 1. 工具概述

课堂行为分析工具是一个统一的 Web 端应用，包含两个核心功能模块：

1. **视频对齐与抽帧**：将前后视角视频按时间对齐，逐秒抽取帧图片
2. **坐姿标注**：对抽帧后的图片进行逐帧标注，包括检测框、状态标签、坐姿标签、姿态关键点

标注结果用于训练"多视角学生行为时序识别模型"，支持课堂坐姿检测和课间行为识别。

### 1.1 启动方式

```bash
python main.py
```

启动后会弹出控制台窗口，浏览器自动打开功能选择页面。关闭控制台窗口即停止服务。

### 1.2 功能选择

打开浏览器后，会看到两个功能卡片：

| 功能 | 说明 | 输入 | 输出 |
|------|------|------|------|
| 视频对齐与抽帧 | 将前后视角视频按时间对齐，逐秒抽取帧图片 | 前后视角视频文件 | forward/backward 目录下的帧图片 |
| 坐姿标注 | 对抽帧后的图片进行逐帧标注 | forward/backward 帧图片 | annotations 目录下的 JSON 标注文件 |

**推荐工作流**：先使用"视频对齐与抽帧"生成帧图片，再使用"坐姿标注"进行标注。

## 2. 视频对齐与抽帧

### 2.1 功能说明

将两个不同视角（前视角和后视角）的课堂监控视频，按各自的第一帧真实时间码进行对齐，然后以每秒 1 帧的频率同步抽取图片，输出到 `forward/` 和 `backward/` 两个文件夹，且严格保证两个视角的帧数一致。

### 2.2 操作流程

```
1. 选择前视角视频文件
      ↓
2. 设置第一帧时间码（时:分:秒）
      ↓
3. 选择后视角视频文件
      ↓
4. 设置第一帧时间码（时:分:秒）
      ↓
5. 选择输出目录
      ↓
6. 可选：选择分辨率限制
      ↓
7. 点击"开始同步抽帧"
      ↓
8. 等待处理完成
```

### 2.3 详细步骤

#### 步骤 1：选择视频

1. 点击"选择"按钮，选择前视角视频文件
2. 点击"选择"按钮，选择后视角视频文件
3. 支持的格式：MP4、AVI、MOV、MKV
4. **注意**：视频文件名不能包含中文

#### 步骤 2：设置时间码

1. 在每个视频下方的时间码输入框中，设置该视频第一帧对应的真实时间
2. 格式为"时:分:秒"，例如 `15:00:00` 表示下午 3 点
3. 工具会自动计算两个视频的时间偏移量

#### 步骤 3：选择输出目录

1. 两个视频都加载后，点击"选择输出目录"按钮
2. 选择一个空目录作为输出位置
3. 工具会在该目录下创建 `forward/` 和 `backward/` 子目录

#### 步骤 4：选择分辨率限制（可选）

1. 如果视频分辨率过高，可以选择降低分辨率以加快处理速度
2. 可选：原分辨率、FHD (1920×1080)、HD (1280×720)、SD (854×480)

#### 步骤 5：开始抽帧

1. 点击"开始同步抽帧"按钮
2. 工具会依次处理两个视频：
   - 根据时间偏移量计算需要跳过的秒数
   - 使用 ffmpeg 逐秒抽取帧图片
   - 自动裁剪尾帧，保证两个视角帧数一致
3. 处理过程中可以点击"停止"按钮中断

### 2.4 输出结果

```
输出目录/
├── forward/          # 前视角帧图片
│   ├── front_000001.jpg
│   ├── front_000002.jpg
│   └── ...
└── backward/         # 后视角帧图片
    ├── back_000001.jpg
    ├── back_000002.jpg
    └── ...
```

- 每秒一帧，文件名按顺序编号
- 前后视角帧数严格一致
- 文件名格式：`front_XXXXXX.jpg` 和 `back_XXXXXX.jpg`

### 2.5 特殊情况处理

#### 损坏视频

如果视频文件的时间戳损坏（常见于某些监控录像），工具会自动检测并使用文件大小估算真实时长，确保抽帧正常进行。

#### 时间偏移

- 如果前视角视频的第一帧时间早于后视角，前视角会跳过开头的部分
- 如果后视角视频的第一帧时间早于前视角，后视角会跳过开头的部分
- 工具会自动计算并显示需要跳过的秒数

## 3. 坐姿标注

### 3.1 功能说明

对抽帧后的图片进行逐帧标注，为每个可见学生标注检测框、状态标签、坐姿标签、姿态关键点等信息。

### 3.2 数据要求

选择的数据文件夹需满足以下条件：

```
数据文件夹/
├── forward/          # 前视角抽帧图片
│   ├── front_000001.jpg
│   ├── front_000002.jpg
│   └── ...
├── backward/         # 后视角抽帧图片
│   ├── back_000001.jpg
│   ├── back_000002.jpg
│   └── ...
└── annotations/      # 标注结果（自动创建）
    ├── front_000001.json
    └── ...
```

- 前后视角图片数量必须一致
- 图片由"视频对齐与抽帧"工具生成
- 文件名不能包含中文

### 3.3 标注内容

#### 每个学生需要标注的字段

| 字段 | 类型 | 说明 |
|------|------|------|
| 检测框 (bbox) | 画框 | 用矩形框框住学生身体，从头顶到臀部 |
| 学生编号 (student_id) | 自动/选择 | 同一学生在整段视频中编号保持一致 |
| 状态标签 (status) | 单选 | 坐-听说 / 坐-读写 / 站立 / 离席 |
| 坐姿标签 (posture) | 单选 | 正常 / 不良 |
| 不良原因 (bad_reason) | 多选 | 歪头 / 低头 / 俯身 / 侧身（仅坐姿=不良时） |
| 前视角姿态点位 | 画点 | 8 个关键点（可选） |
| 后视角姿态点位 | 画点 | 8 个关键点（可选，需先画后视角框） |
| 备注 (note) | 文本 | 可选，特殊情况说明 |

#### 状态标签定义

| 状态 | 含义 | 典型场景 |
|------|------|----------|
| 坐-听说 | 坐着听老师讲课 | 老师讲授、学生抬头看黑板 |
| 坐-读写 | 坐着读书或写字 | 学生低头看书、写字 |
| 站立 | 站着 | 回答问题、站起来活动 |
| 离席 | 不在座位上 | 离开教室、在教室走动 |

#### 坐姿标签定义

| 坐姿 | 含义 |
|------|------|
| 正常 | 坐姿端正，头部和身体基本保持正直 |
| 不良 | 存在明显的不良坐姿表现 |

#### 不良原因定义

当坐姿标记为"不良"时，需要勾选具体原因：

| 原因 | 含义 | 典型表现 |
|------|------|----------|
| 歪头 | 头部向左或右倾斜 | 头部不正，偏向一侧 |
| 低头 | 头部向下低垂 | 头部低于正常位置，常见于看手机或打瞌睡 |
| 俯身 | 上身向前弯曲 | 上半身趴在桌上或严重前倾 |
| 侧身 | 身体向左或右倾斜 | 身体不正，偏向一侧 |

#### 姿态点位定义

8 个上半身关键点，按标注顺序：

| 序号 | 名称 | 位置 | 标注要点 |
|------|------|------|----------|
| 1 | 头 | 头顶正中 | 发际线附近 |
| 2 | 脖子 | 颈部与躯干连接处 | 颈部根部 |
| 3 | 右肩 | 右侧肩关节 | 肩膀最外侧 |
| 4 | 右肘 | 右侧肘关节 | 手臂弯曲处 |
| 5 | 右手 | 右侧手腕或手掌 | 手腕位置 |
| 6 | 左肩 | 左侧肩关节 | 肩膀最外侧 |
| 7 | 左肘 | 左侧肘关节 | 手臂弯曲处 |
| 8 | 左手 | 左侧手腕或手掌 | 手腕位置 |

骨架连接关系：
```
        头
         │
        脖子
       ╱    ╲
   右肩      左肩
      │        │
   右肘      左肘
      │        │
   右手      左手
```

**注意**：
- 被遮挡的点可以跳过（标记为不可见）
- 前视角看不到的点，可以在后视角标注（需先画后视角框）
- 前后视角的点位独立存储，不会跨视角连线

### 3.4 标注流程

#### 整体流程

```
1. 选择数据文件夹
      ↓
2. 加载第一帧图片
      ↓
3. 对当前帧中每个可见学生依次标注：
   ① 画框 → ② 调整编号 → ③ 填写标签 → ④ 标注点位（可选）
      ↓
4. 所有学生标注完毕 → 点击"下一帧"或按 → 键
      ↓
5. 重复步骤 3-4，直到所有帧标注完成
      ↓
6. 标注数据自动保存到 annotations/ 目录
```

#### 单个学生的标注步骤

**步骤 1：画框**

1. 在前视角画面上，按住鼠标左键拖拽，画出一个矩形框
2. 框的范围：从头顶到臀部下方，尽量贴合身体轮廓
3. 松开鼠标后，系统自动分配编号（S01、S02...），并弹出标签面板

**步骤 2：填写标签**

标签面板会自动弹出，所有修改即时生效，无需点击确认：

1. **选择状态**：点击"坐-听说"、"坐-读写"、"站立"或"离席"
2. **选择坐姿**：点击"正常"或"不良"
3. **选择不良原因**（仅坐姿=不良时）：勾选"歪头"、"低头"、"俯身"、"侧身"
4. **填写备注**（可选）：输入特殊情况说明

**步骤 3：标注姿态点位（可选）**

1. 点击标签面板中的"前视角"按钮，或按 `K` 键进入点位标注模式
2. 按顺序点击放置 8 个关键点：头 → 脖子 → 右肩 → 右肘 → 右手 → 左肩 → 左肘 → 左手
3. 被遮挡的点：右键点击跳过
4. 点击框外区域或按 `Esc` 退出点位标注模式
5. 如需在后视角标注：
   - 先在后视角画一个框（选中学生后，在后视角拖拽画框）
   - 点击标签面板中的"后视角"按钮，或按 `Shift+K` 键
   - 在后视角框内标注点位

**步骤 4：处理下一个学生**

1. 在前视角空白区域点击，取消当前选中
2. 重复步骤 1-3，标注下一个学生

### 3.5 高效标注技巧

#### 复制上一帧（核心效率功能）

课堂场景中，学生位置和姿态变化缓慢。大部分帧可以直接复制上一帧的标注，然后微调有变化的学生。

1. 按 `C` 键，或点击"复制上帧"按钮
2. 上一帧的所有标注（框 + 标签 + 点位）会复制到当前帧
3. 检查是否有学生位置或状态变化，有变化的进行微调

#### 批量处理流程

1. 标注第 1 帧（最慢，需要画所有框）
2. 按 → 切换到第 2 帧
3. 按 `C` 复制上一帧
4. 检查并微调有变化的学生
5. 重复步骤 2-4

## 4. 注意事项

### 4.1 标注规范

1. **框要贴合**：画框时尽量贴合学生身体轮廓，不要太大或太小
2. **编号一致**：同一学生在整段视频中使用相同编号，不要重复或遗漏
3. **状态准确**：根据学生当前动作准确选择状态标签
4. **不良原因要选**：标记为"不良"时，必须勾选至少一个不良原因
5. **点位可选**：姿态点位是可选的，根据需要标注

### 4.2 常见问题

**Q: 标注过程中 annotations 文件夹被删除了怎么办？**
A: 工具会自动重建文件夹，继续保存当前帧。但之前已标注的帧数据会丢失，帧列表会自动同步状态。

**Q: 后视角的框和点位怎么不显示？**
A: 后视角框和点位只在选中前视角对应框时才显示。点击前视角框即可显示。

**Q: 如何修改已标注学生的编号？**
A: 双击框打开标签面板，在"学生编号"下拉框中选择新编号。如果新编号已被其他学生使用，会自动交换两个学生的编号。

**Q: 点位标注时标错了怎么办？**
A: 按 `Ctrl+Z` 撤销上一个点位，可连续多次撤销。按 `Ctrl+Y` 重做。

**Q: 如何只标注部分点位？**
A: 可以只标注可见的点，被遮挡的点右键跳过。也可以只在前视角或后视角标注，不要求两个视角都标。

**Q: 视频文件名包含中文怎么办？**
A: 抽帧工具会提示错误，请先将视频文件名改为英文或数字。

**Q: 抽帧过程中想停止怎么办？**
A: 点击"停止"按钮即可中断抽帧过程。

### 4.3 效率建议

1. **先标第 1 帧**：第 1 帧需要画所有框，是最慢的一帧
2. **大量使用复制**：第 2 帧开始，按 `C` 复制上一帧，只调整有变化的学生
3. **善用快捷键**：`K` 标点位、`Del` 删除、`Ctrl+Z` 撤销
4. **后视角辅助**：前视角遮挡的学生，参考后视角来判断状态和坐姿
5. **点位按需标注**：不是每个学生都需要标点位，根据需要选择性标注

## 6. 运行方式

### 6.1 统一入口（推荐）

```bash
python main.py
```

启动后浏览器自动打开功能选择页面，可以选择进入抽帧工具或标注工具。

### 6.2 独立入口

如果只需要使用其中一个工具，也可以直接运行对应的入口文件：

```bash
# 只使用抽帧工具
python web_app.py

# 只使用标注工具
python annotation_app.py
```

### 6.3 打包为可执行文件

抽帧工具支持打包为独立的 exe 文件，无需安装 Python 环境即可运行：

```bash
build.bat
```

打包完成后，生成的可执行文件位于 `dist/web_app.exe`。
