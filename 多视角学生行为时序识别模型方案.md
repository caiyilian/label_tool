# 多视角学生行为时序识别模型方案

## 1. 目的与范围

本方案仅面向当前初级目标：

1. 课堂坐姿检测
2. 课间行为识别

本方案只整理两部分内容：

1. 需要人工标注的标签
2. 模型的输入与输出

---

## 2. 核心标注单位

建议统一采用以下标注单位：

> 一个样本 = 某班级、某一秒、某一个学生、前后摄像头对应画面下的标注结果

即：

```text
class_id
+ real_timestamp
+ student_id
+ front_frame / back_frame
+ 该秒该学生的检测框、可见性、坐姿或课间行为标签
```

说明：

1. 这里的核心是“每秒、每学生、双视角对应标注”。
2. 班级级统计结果后续可由学生级输出自动汇总，不需要人工单独标注。
3. 视频很长，没一帧都标注的话工作量太大而且也没必要，所以只标注每一秒，因为学生在1秒内通常不会发生显著动作变化

---

## 3. 需要标注的标签

## 3.1 时间与同步标签

| 标签               | 标注单位 | 说明                 | 示例                        |
| ------------------ | -------: | -------------------- | --------------------------- |
| `class_id`       |   视频级 | 班级编号             | `Grade7_Class02`          |
| `date`           |   视频级 | 拍摄日期             | `2026-04-01`              |
| `real_timestamp` |     每秒 | 真实时间             | `2026-04-01 09:31:24`     |
| `front_frame_id` |     每秒 | 前摄像头对应帧编号   | `front_012345`            |
| `back_frame_id`  |     每秒 | 后摄像头对应帧编号   | `back_012347`             |
| `period_state`   |     每秒 | 当前处于上课还是课间 | `class` / `recess`      |
| `lesson_index`   | 时间段级 | 第几节课或第几次课间 | `lesson_3` / `recess_3` |

要求：

```text
前后摄像头在同一真实时间点应能对应到同一秒的画面。
```

这些字段属于样本组织与时间同步元数据，主要用途如下：

1. `class_id`：区分不同班级的数据归属。
2. `date`：区分不同日期的数据，便于按天整理和后续趋势统计。
3. `real_timestamp`：作为统一时间轴，保证前后视角对应到同一时刻。
4. `front_frame_id`、`back_frame_id`：定位该秒对应的具体视频帧，便于回溯和校验。
5. `lesson_index`：区分第几节课或第几次课间，便于按课次汇总。

---

## 3.2 学生目标与可见性标签

| 标签                | 标注单位 | 说明                     | 示例                                                       |
| ------------------- | -------: | ------------------------ | ---------------------------------------------------------- |
| `student_id`      |   学生级 | 匿名学生唯一编号         | `C02_S017`                                               |
| `front_visible`   |     每秒 | 前摄像头下是否可见       | `visible` / `partial` / `occluded` / `out_of_view` |
| `back_visible`    |     每秒 | 后摄像头下是否可见       | `visible` / `partial` / `occluded` / `out_of_view` |
| `front_bbox`      |     每秒 | 前摄像头下该学生的人体框 | `[480, 220, 535, 390]`                                   |
| `back_bbox`       |     每秒 | 后摄像头下该学生的人体框 | `[910, 250, 990, 430]`                                   |
| `occlusion_level` |     每秒 | 当前遮挡程度             | `none` / `mild` / `severe`                           |

说明：

1. 课堂阶段不单独标注 `position_zone`，因为正常情况下默认都处于 `seat_area`。
2. 保留 `occlusion_level`，可用于辅助判断该秒模型输出结果的可信度。

---

## 3.3 课堂坐姿标签

| 标签                 |   标注单位 | 说明                   | 取值                                                                                                        |
| -------------------- | ---------: | ---------------------- | ----------------------------------------------------------------------------------------------------------- |
| `posture_overall`  | 每秒每学生 | 总体坐姿               | `standard` / `bad`                                                                                      |
| `posture_severity` | 每秒每学生 | 不良程度               | `normal` / `mild` / `severe`                                                                          |
| `head_status`      | 每秒每学生 | 头部状态               | `upright` / `head_down` / `forward` / `tilt_left` / `tilt_right` / `turn_left` / `turn_right` |
| `torso_status`     | 每秒每学生 | 上身状态               | `upright` / `slight_forward` / `severe_forward` / `desk_prone` / `side_prone`                     |
| `body_balance`     | 每秒每学生 | 身体平衡状态           | `balanced` / `lean_left` / `lean_right` / `twisted`                                                 |
| `shoulder_balance` | 每秒每学生 | 肩部平衡状态           | `balanced` / `left_low` / `right_low`                                                                 |
| `learning_action`  | 每秒每学生 | 当前学习动作           | `listening` / `writing` / `reading` / `resting` / `talking`                                       |
| `bad_posture_type` | 每秒每学生 | 不良坐姿子类型，可多选 | `head_down` / `forward_lean` / `desk_prone` / `body_tilt` / `side_prone`                          |
| `posture_valid`    | 每秒每学生 | 该秒是否适合判断坐姿   | `valid` / `invalid_due_to_occlusion` / `invalid_due_to_absence`                                       |

说明：

1. 若确实因遮挡或学生不在画面中无法判断，通过 `posture_valid` 标为无效。
2. 保留 `learning_action`，避免把正常写字低头全部误判为不良坐姿。

---

## 3.4 课间行为标签

| 标签               |   标注单位 | 说明                     | 取值                                                                                                          |
| ------------------ | ---------: | ------------------------ | ------------------------------------------------------------------------------------------------------------- |
| `presence_state` | 每秒每学生 | 学生是否在教室内         | `in_classroom` / `left_classroom`                                                                         |
| `seat_state`     | 每秒每学生 | 是否仍在自己座位附近     | `at_seat` / `away_from_seat`                                                                              |
| `activity_state` | 每秒每学生 | 当前活动状态             | `sitting` / `standing` / `walking` / `running` / `lying` / `playing` / `entering` / `leaving` |
| `movement_level` | 每秒每学生 | 活动强度                 | `static` / `low` / `medium` / `high`                                                                  |
| `position_zone`  | 每秒每学生 | 当前所在区域             | `seat_area` / `aisle` / `door` / `podium` / `out_of_room`                                           |
| `leave_event`    | 每秒每学生 | 该秒是否发生离开教室事件 | `leave_classroom` / `None`                                                                                |
| `enter_event`    | 每秒每学生 | 该秒是否发生进入教室事件 | `enter_classroom` / `None`                                                                                |
| `recess_valid`   | 每秒每学生 | 该秒是否适合判断课间行为 | `valid` / `invalid_due_to_occlusion`                                                                      |

说明：

1. `leave_event` 和 `enter_event` 是事件标签，不是持续状态标签。
2. 如果某一秒该学生没有发生离开教室动作，则 `leave_event = None`。
3. 如果某一秒该学生没有发生进入教室动作，则 `enter_event = None`。
4. 如果同一秒这两个动作都没有发生，则二者都标为 `None`。
5. `position_zone` 仅在课间阶段标注，用于描述学生在教室内外的活动区域。

## 5. 模型输入与输出

## 5.1 先说明：哪些是输入，哪些是监督标签

本项目里，医生或标注员标出来的内容，不应全部视为模型输入。

应区分为两类：

### A. 模型输入

模型真正看到的是视频数据。

### B. 监督标签

人工标注的学生框、坐姿标签、课间行为标签，用于训练时计算损失，不是部署时必须人工提供的输入。

这是因为该模型最终要用于未标注视频，不能依赖人工先把学生位置或行为标出来再喂给模型。

---

## 5.2 训练阶段

### 训练输入

```text
前摄像头连续视频片段
+ 后摄像头连续视频片段
+ period_state
```

可选补充信息：

```text
课程表或上课/课间时间表
```

### 训练监督标签

```text
student_id
front_bbox
back_bbox
front_visible
back_visible
occlusion_level
posture_overall
posture_severity
head_status
torso_status
body_balance
shoulder_balance
learning_action
bad_posture_type
posture_valid
presence_state
seat_state
activity_state
movement_level
position_zone
leave_event
enter_event
recess_valid
```

结论：

> 学生实时位置属于模型训练时的监督信息，不应写成模型部署时必须提供的输入；`occlusion_level` 可作为辅助输出，用于提示当前结果受遮挡影响的程度。

---

## 5.3 推理阶段

### 推理输入

```text
前摄像头同步视频
+ 后摄像头同步视频
+ 可选的上课/课间时间表
```

### 推理输出

模型应直接从视频中输出：

1. 每秒每个学生的位置、可见性和遮挡信息
2. 每秒每个学生的课堂坐姿结果
3. 每秒每个学生的课间行为结果

---

## 5.4 课堂输出

上课时，模型对每秒每学生输出：

| 输出项               | 说明            |
| -------------------- | --------------- |
| `student_id`       | 匿名学生编号    |
| `real_timestamp`   | 时间戳          |
| `front_bbox`       | 前视角检测框    |
| `back_bbox`        | 后视角检测框    |
| `front_visible`    | 前视角可见性    |
| `back_visible`     | 后视角可见性    |
| `occlusion_level`  | 遮挡程度        |
| `posture_overall`  | 标准 / 不良坐姿 |
| `posture_severity` | 不良程度        |
| `head_status`      | 头部状态        |
| `torso_status`     | 上身状态        |
| `body_balance`     | 身体平衡状态    |
| `shoulder_balance` | 肩部平衡状态    |
| `learning_action`  | 当前学习动作    |
| `bad_posture_type` | 不良坐姿子类型  |
| `posture_valid`    | 本秒是否有效    |

---

## 5.5 课间输出

课间时，模型对每秒每学生输出：

| 输出项              | 说明                     |
| ------------------- | ------------------------ |
| `student_id`      | 匿名学生编号             |
| `real_timestamp`  | 时间戳                   |
| `front_bbox`      | 前视角检测框             |
| `back_bbox`       | 后视角检测框             |
| `front_visible`   | 前视角可见性             |
| `back_visible`    | 后视角可见性             |
| `occlusion_level` | 遮挡程度                 |
| `presence_state`  | 是否在教室内             |
| `seat_state`      | 是否在座位附近           |
| `activity_state`  | 当前活动状态             |
| `movement_level`  | 活动强度                 |
| `position_zone`   | 当前所在区域             |
| `leave_event`     | 本秒是否发生离开教室事件 |
| `enter_event`     | 本秒是否发生进入教室事件 |
| `recess_valid`    | 本秒是否有效             |

---

## 5.6 班级统计结果如何得到

班级级结果不需要单独人工标注。

可直接由每秒每学生输出自动汇总得到，例如：

1. 每节课班级标准坐姿率、不良坐姿率
2. 每天班级坐姿变化趋势
3. 每节课间班级离班率
4. 每天班级课间活动趋势

因此，班级级统计结果应视为后处理汇总结果，而不是单独标签。

## 附录：标签中文解释

### 1. 区域类标签

| 标签            | 中文解释                                           |
| --------------- | -------------------------------------------------- |
| `seat_area`   | 座位区域，指学生自己座位及紧邻座位附近的小范围区域 |
| `aisle`       | 过道区域，指课桌之间或课桌与墙边之间可供通行的区域 |
| `door`        | 门口区域，指靠近教室出入口的位置                   |
| `podium`      | 讲台区域，指教师讲台及其紧邻区域                   |
| `out_of_room` | 教室外，指学生已经离开教室画面对应空间             |

### 2. 可见性标签

| 标签            | 中文解释                                               |
| --------------- | ------------------------------------------------------ |
| `visible`     | 清晰可见，学生主体基本可以判断                         |
| `partial`     | 部分可见，学生只有部分身体出现在画面中，但仍可辅助判断 |
| `occluded`    | 被遮挡，学生被桌椅、书本或其他学生明显遮挡，难以判断   |
| `out_of_view` | 不在该摄像头视野内                                     |

### 3. 遮挡程度标签

| 标签       | 中文解释                       |
| ---------- | ------------------------------ |
| `none`   | 无明显遮挡                     |
| `mild`   | 轻度遮挡，仍可大致判断学生状态 |
| `severe` | 重度遮挡，难以可靠判断学生状态 |

### 4. 事件标签（某个学生）

| 标签                              | 中文解释                 |
| --------------------------------- | ------------------------ |
| `leave_event = leave_classroom` | 该秒发生“离开教室”事件 |
| `enter_event = enter_classroom` | 该秒发生“进入教室”事件 |
| `leave_event = None`            | 该秒没有发生离开教室事件 |
| `enter_event = None`            | 该秒没有发生进入教室事件 |
